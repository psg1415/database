데이터베이스와 데이터베이스 시스템
데이터 정의 - DDL(Data Definition Language) - 데이터 정의어
           - 일관성(자료형, 제약조건)
데이터 변경 - DML(Data Manipulation Language) - 데이터 조작어

데이터베이스 시스템
1. 데이터베이스  - 데이터
2. DBMS - 소프트웨어
3. 데이터모델 - 데이터가 저장되는 기법

데이터베이스 언어
1. 데이터 정의어(DDL) - 자료형, 제약조건...
2. 데이터 조작어(DML) - 데이터를 조회, 변경, 수정, 삭제
3. 데이터 제어어(DCL) - 데이터의 사용자 권한






3단계 데이터베이스 구조
외부 스키마 - 전체 데이터 중 특정 그룹에서만 필요한 데이터의 구조
개념 스키마 - 전체 데이터에 대한 구조
내부 스키마 - 저장장치에 실제로 저장되는 방법
              (자료형, 제약조건 ...)

스키마 - 데이터의 구조(자료형 + 제약조건...)


데이터 독립성
1) 논리적 데이터 독립성
개념 스키마 -> 외부 스키마에 영향 X


2) 물리적 데이터 독립성
내부 스키마(저장장치 관련 - 물리적) -> 개념 스키마에 영향 X

관계 데이터 모델
1. 수학의 집합 이론에 근거
2. 비절차적 언어

SELECT 컬럼...(3) FROM 테이블(1) WHERE 조건(2)

릴레이션 - 행과 열로 구성된 테이블
        - (테이블)
        - 수학의 집합에서 나온 개념









관계
  1. 릴레이션 내에서 생성되는 관계
  2. 릴레이션 간에 생성되는 관계
      (테이블 - 릴레이션 간의 공통 조건에 따른 관계)

스키마 - 데이터의 구조(데이터가 저장되는 방식 - 자료형, 제약조건)
       - DDL(CREAT DATABASE, CREAT TABLE ...)

       - 속성(칼럼) - 각각 항목
       - 도메인 - 속성이 가질 수 있는 값의 집합
                  (자료형, 제약조건)
       - 차수 - 속성의 갯수
       - 카디날리티 - 행(튜플)의 갯수

인스턴스 - 실제로 스키마에 저장된 데이터

튜플 - 각각의 데이터 행(row)

차수 - 컬럼(열)의 총 갯수

카디날리티 - 행(튜플)의 총 갯수
